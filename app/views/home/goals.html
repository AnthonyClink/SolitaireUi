Steve, Anthony - Why did you decide to build Solitaire?

We are attempting to build a fluid rest platform jam packed full of features for data analysis with some fun easy APIs for handling complex data structures from a web service.

For both Steve and I, this is all new. I am a professional developer, but have never created something truly from scratch.
We are attempting to create a platform that is completely Java SE compliant
The features we hope to include in the overall platform are:

  • No XML configuration beyond a few simple POM's and a very trivial web.xml file
  • No complex wiring of common services
  • Fully testable complete with all current test driven methodologies
  • Complete separation between the ui and the API for ClinkWorks product
  • Out of the box polygot database support
  • Generated UI
  • Annotation based event API
  • Annotation based business rule engine
  • Annotation based API security administration
We are going to attempt to accomplish this feat utilizing as close to pure jsr standard API's to create very simple platform with a rich set of standards compliant features to allow easy creation of complex user centric data management systems. systems.

For those of you interested, here is an overview of the overall architecture for the platform:

Server

  • RESTful services are provided through the JAX-RS API through the reference implementation Jersey
  • Object to relational data mapping provided through the Java Persistence API via the reference implementation Apache OpenJPA
  • Object serialization/deserialization provided through the JAXB API via the Jackson Object Serialization API
  • Configuration and dependency management provided through JSR-299 via Google Guice
  • Collections management provided by Google Guava

Client

  • MVC provided by Angular
  • Object serialization/deserliazation provided through Angular.$Resource
  • Data binding provided by Angular
  • REST API management provided by Angular.$Resource
  • Collections management provided by LoDash via a custom angular service
  • REST/UI management done via a custom data api
