Goals:

We are attempting to build a fluid rest platform jam packed full of features for data analysis with some fun easy APIs for push notifications and customer data gathering.

For both Steve and I, this is a new endevor. We are attempting to create a platform that is completly java se compliant utilizing as close to pure jsr standard API's to create a rich set of features to allow easy creation of complex data gathering systems.

For those of you interested, heres the overall archetecture for the platform:

Server

  • RESTFul java api using the JAX-RS API through the implementation Jersey
  • Object to relational data mapping provided through the Java Persistence API via Apache OpenJPA
  • Object serialization/desearlization provided through the JAXB API via Jackson
  • Configuration and dependency management provided through JSR-299 via Google Guice
  • Collections management provided by Google Guava

Client

  • MVC provided by Angular
  • Object serialization/deserliazation provided through Angular.$Resource
  • Data binding provided by Angular
  • REST API management provided by Angular.$Resource
  • Collections management provided by LoDash via a custom angular service
  • REST/UI management done via a custom data api
