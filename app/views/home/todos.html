    For those of you interested, here is an overview of the overall architecture for the platform:

    Server

    • RESTful services are provided through the JAX-RS API through the reference implementation Jersey
    • Object to relational data mapping provided through the Java Persistence API via the reference implementation Apache OpenJPA
    • Object serialization/deserialization provided through the JAXB API via the Jackson Object Serialization API
    • Configuration and dependency management provided through JSR-299 via Google Guice
    • Collections management provided by Google Guava
    • Build Management - Maven
    • Continuous Build Server - Jenkins
    • Project management and task tracking Kanbanize (free and awesome btw - lacking features but has enough for a 2 man team)
    • Project hosting by Amazon EC2 Cloud
    • Development Server - Jetty
    • Development Database - H2
    • Production Server - Tomcat
    • Production Database - PostgreSQL
    • TDD software - JUnit
    • Mock framework - JMockIt

    Client

    • Production Server - Apache Server
    • Test Server - Karma
    • MVC provided by Angular
    • Object serialization/deserialization provided through Angular.$resource
    • Data binding provided by Angular
    • REST API management provided by Angular.$resource
    • Collections management provided by LoDash via a ClinkWorks Service
    • SERVER/UI Data Management done via a ClinkWorks domain model
    • Build Management - Grunt
    • TDD software - Jasmine
    • Dependency Management - NPM
    • Mock Framework - Angular.$mock
